회선교환망 : 데이터를 설정된 통신 회선을 통하여 전송하는 방식으로서 정보량이 많을 때와 파일 전송 등의 긴 메시지 전송에 적합하여 정보전송의 필요성이 생겼을 때 상대방을 호줄하여 연결하고 이 물리적인 연결이 정보 전송이 종료필 때 까지 계속 유지되는 망은 무엇인가?

회선교환 방식 : 음성 전화망과 같이 메시지가 전송되기 전에 발생지에서 목적지까지의 물리적 통신 회선 연결이 선행되어야 하는 교환 방식은?

네트워크 표준안 : 토큰링 방식 : IEEE 802.5

RTCP (real-time control protocol) : session의 모든 참여자에게 컨트롤 패킷을 주기적으로 전송한다.
				(X)RTCP 패킷은 항상 16비트의 경계로 끝난다.
				하위 프로토콜은 데이터 패킷과 컨트롤 패킷의 멀티플렉싱을 제공한다.
				데이터 전송을 모니터링하고 최소한의 제어와 인증 기능을 제공한다.
DHCP : 각 컴퓨터에서 IP 관리를 쉽계 하기 위한 프로토콜이며, TCP/IP 통신을 쇧행하기 위해 필요한 정보를 자동적으로 할당, 관리하기 위한 통신 규약으로써 RFC 1541에 규정되어 있다.

네트워크 구성 형태 (topology) : 성형 (star) : 중앙에 호스트 컴퓨터가 있고 이를 중심으로 터미널들이 연결되는 네트워크 구성 형태

scrambler : 데이터통신 시스템에서 동기데이터 암호화를 통해 데이터베이스 관리시스템의 액세스를 막는 장치로 주로 원거리 전송에 이용되는 것은?

여러 개의 통신 채널이 존재하는 FDM (주파수 분할 다중화 방식) : 전송에 있어 시간의 지연 없이 실시간 전송을 한다.
							채널간의 상호 간섭을 막기 위해 완충지역으로 보호대역이 필요하다.
							주파수 분할은 모뎀의 역할을 겸하므로 별도의 모뎀을 필요로 하지 않는다.
							(X)동기 전송에서 사용된다.
